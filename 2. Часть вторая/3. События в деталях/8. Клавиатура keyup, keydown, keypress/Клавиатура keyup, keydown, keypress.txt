1. Не стоит ставить обработчики на Caps Lock
2. Для проверки в полях ввода лучше ни использовать события ниже

keydown и keyup - происходят при нажатии/отпускании клавиши
    получить скан-код клавиши в свойстве e.keyCode
        для букв и цифр код клавиши будет равен коду соответствующей заглавной английской буквы/цифры. "S".charCodeAt(0).
        для других клавиш стенд: https://learn.javascript.ru/keyboard-events#keyboard-test-stand
        исключения:
            Клавиша	    Firefox	    Остальные браузеры
            ;	        59	        186
            =	        107	        187
            -	        109	        189
    Для реализации горячих клавиш, включая сочетания – используем keydown






keypress - возникает сразу после keydown, если нажатие приводит к появлению символа (буквы, цифры, не Ctrl, Shift).
    получить код символа получить код символа charCode и e.which
    Кросс-браузерная функция для получения символа из события keypress:
    // event.type должен быть keypress
    function getChar(event) {
      if (event.which == null) { // IE
        if (event.keyCode < 32) return null; // спец. символ
        return String.fromCharCode(event.keyCode)
      }

      if (event.which != 0 && event.charCode != 0) { // все кроме IE
        if (event.which < 32) return null; // спец. символ
        return String.fromCharCode(event.which); // остальные
      }

      return null; // спец. символ
    }

-------------------------------------------Отмена пользовательского ввода ------------------------------------------
Появление символа можно предотвратить, если отменить действие браузера на keydown/keypress
Отменять можно не только символ, а любое действие клавиш которые происходят на уровне операционной системы
